Dimensionality Reduction
§1 · What is PCA?
IntuitionUnsupervised
🧭 Core Idea

PCA finds the directions of maximum variance in high-dimensional data, then projects the data onto fewer dimensions — losing as little information as possible.

  • Reduces p features → k components (k << p)
  • New axes = principal components (PCs)
  • PCs are orthogonal to each other
  • First PC captures most variance, second the next, etc.
  • Unsupervised — ignores labels completely
Original data: n samples × p features
After PCA: n samples × k components
where k ≤ p (usually k << p)
Use CasesWhen to apply
✅ When to Use PCA
  • 📉 Curse of dimensionality — too many features
  • 🧹 Remove multicollinearity before regression/ML
  • 👁️ Visualize high-dim data in 2D/3D
  • ⚡ Speed up model training (fewer features)
  • 🗜️ Data compression & storage
  • 🔬 Noise reduction in signals/images
  • 🧬 Genomics, image recognition, finance
❌ Avoid when: interpretability matters
(PCs are linear combos — hard to explain)
❌ Non-linear structure → use UMAP / t-SNE
GlossaryKey Terms
📖 Key Terms
  • Principal Component: new axis of max variance
  • Loading vector: direction of a PC in original space
  • Score: projection of a sample onto a PC
  • Eigenvalue: variance explained by each PC
  • Eigenvector: direction of each PC
  • Explained variance ratio: fraction of total variance per PC
  • Scree plot: eigenvalue vs PC number
  • Biplot: scores + loadings in same plot
§2 · The Mathematics
AlgorithmStep by Step
🔢 PCA Algorithm — 6 Steps
1
Standardize the data — subtract mean, divide by std. Each feature gets μ=0, σ=1. (Critical if features have different scales)
2
Compute Covariance Matrix Σ — a p×p matrix capturing how each pair of features varies together.
3
Eigen-decomposition — find eigenvalues λ and eigenvectors v of Σ. Each eigenvector is a PC direction.
4
Sort eigenvalues in descending order. Largest λ = most variance = PC1.
5
Select k components — choose how many PCs to keep using explained variance (e.g. 95% threshold or elbow in scree plot).
6
Project data onto selected PCs — multiply standardized data X by the k eigenvectors → new k-dim representation.
FormulasCore Math
∑ Key Equations
Standardize
z = (x − μ) / σ
Covariance Matrix
Σ = (1/n) Xᵀ X
Σᵢⱼ = Cov(xᵢ, xⱼ)
Eigen-decomposition
Σ · v = λ · v
v = eigenvector (PC)
λ = eigenvalue (variance)
Projection
Z = X · W
W = matrix of k eigenvectors
Z = scores (new coordinates)
Explained Variance
ratio = λᵢ / Σλⱼ
§3 · PCA with Python & scikit-learn
Setupsklearn
📦 Import & Fit PCA
from sklearn.preprocessing import StandardScaler
from sklearn.decomposition import PCA
import numpy as np
import pandas as pd

# 1. Standardize
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

# 2. Fit PCA
pca = PCA(n_components=2)
X_pca = pca.fit_transform(X_scaled)

# Shape: (n_samples, 2)
print(X_pca.shape)
AttributesAfter fitting
🔍 Key Attributes
# Explained variance ratio per PC
pca.explained_variance_ratio_
# e.g. [0.72, 0.15, 0.08, ...]

# Cumulative explained variance
np.cumsum(pca.explained_variance_ratio_)

# The principal components (loadings)
pca.components_ # shape (k, p)

# Eigenvalues
pca.explained_variance_

# Singular values
pca.singular_values_

# Mean of each feature
pca.mean_
SelectionHow many PCs?
🎯 Choosing n_components
# Option 1: fixed number
pca = PCA(n_components=3)

# Option 2: % of variance (e.g. 95%)
pca = PCA(n_components=0.95)

# Option 3: keep all, then inspect
pca = PCA() # no n_components
pca.fit(X_scaled)
cumvar = np.cumsum(
    pca.explained_variance_ratio_)
k = np.argmax(cumvar >= 0.95) + 1
print(f"Need {k} PCs for 95% var")
Reconstruction
🔄 Inverse Transform

Project back to original space — useful for measuring reconstruction error or denoising.

# Reconstruct original features
X_reconstructed = pca.inverse_transform(X_pca)

# Reconstruction error (MSE)
error = np.mean((X_scaled - X_reconstructed)**2)
Best Practice
🔗 sklearn Pipeline

Chain StandardScaler + PCA + model in a single reusable pipeline.

from sklearn.pipeline import Pipeline
from sklearn.svm import SVC

pipe = Pipeline([
  ('scale', StandardScaler()),
  ('pca', PCA(n_components=0.95)),
  ('clf', SVC())
])
pipe.fit(X_train, y_train)
Large Data
⚡ Incremental PCA

For large datasets that don't fit in memory — processes data in mini-batches.

from sklearn.decomposition import IncrementalPCA

ipca = IncrementalPCA(n_components=50,
                     batch_size=200)

for batch in np.array_split(X, 10):
    ipca.partial_fit(batch)

X_reduced = ipca.transform(X)
Non-Linear
🌀 Kernel PCA

Extends PCA to non-linear structure using the kernel trick. Good for complex manifolds.

from sklearn.decomposition import KernelPCA

# RBF kernel (Gaussian)
kpca = KernelPCA(
    n_components=2,
    kernel='rbf',
    gamma=0.04
)
X_kpca = kpca.fit_transform(X_scaled)
# kernels: rbf, poly, sigmoid, cosine
§4 · Visualization
Scree Plotmatplotlib
📉 Scree Plot — Explained Variance

Visualize how much variance each component explains. Look for the "elbow" to choose k.

import matplotlib.pyplot as plt

pca_full = PCA().fit(X_scaled)
evr = pca_full.explained_variance_ratio_

fig, ax = plt.subplots(1, 2, figsize=(10,4))

# Bar chart
ax[0].bar(range(1, len(evr)+1), evr)
ax[0].set_xlabel('PC')
ax[0].set_ylabel('Explained Variance')

# Cumulative line
ax[1].plot(np.cumsum(evr), 'o-')
ax[1].axhline(0.95, ls='--', color='red')
ax[1].set_ylabel('Cumulative Var')

🔵 2D PCA Scatter Plot

Project to 2D and color by class label — classic way to inspect cluster separation.

import seaborn as sns

pca2 = PCA(n_components=2)
coords = pca2.fit_transform(X_scaled)

df_pca = pd.DataFrame(
    coords,
    columns=['PC1', 'PC2']
)
df_pca['label'] = y

sns.scatterplot(
    data=df_pca,
    x='PC1', y='PC2',
    hue='label', palette='Set2'
)

§5 · Tips, Pitfalls & Alternatives
Best Practices
💡 Tips & Tricks
  • Always standardize first — PCA is scale-sensitive
  • Fit scaler & PCA on train set, apply to test
  • Use 95% cumulative variance as default threshold
  • Check the elbow in scree plot visually
  • PCA on correlation matrix = standardized data
  • Inspect loadings to understand what PCs mean
Watch Out
⚠️ Common Pitfalls
  • ❌ Don't apply PCA before scaling — results are meaningless
  • ❌ Don't fit PCA on the test set (data leakage!)
  • ❌ PCA assumes linear relationships only
  • ❌ Outliers strongly distort PCA — clean first
  • ❌ PCA removes noise AND signal — can hurt accuracy
  • ❌ PCs have no intuitive meaning (not original features)
Alternatives
🔀 PCA vs Alternatives
  • t-SNE: non-linear, visualization only (no transform)
  • UMAP: non-linear, faster, preserves global structure
  • LDA: supervised — maximizes class separation
  • Autoencoder: non-linear deep learning version of PCA
  • ICA: finds statistically independent components
  • NMF: non-negative features (images, text)
